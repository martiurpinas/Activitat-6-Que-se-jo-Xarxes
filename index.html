Classificación de IPs

Classes y rangos importants

  • Classe A: 1.0.0.0 – 126.255.255.255 (grans xarxes, fins a 16 milions d'hosts).
  • Classe B: 128.0.0.0 – 191.255.255.255 (xarxes mitjanes, fins a 65.000 hosts).
  • Classe C: 192.0.0.0 – 223.255.255.255 (xarxes petites, fins a 254 hosts).

Rangos privados (úso habitual)

  • 10.0.0.0/8: Utilitzat en grans empreses per a xarxes internes.
  • 172.16.0.0/12: Ideal per a xarxes mitjanes o departamentals.
  • 192.168.0.0/16: Comú en xarxes domèstiques i petites oficines.

Les adreces privades no són accessibles des d'Internet i requereixen NAT per a connexions externes. Això garanteix seguretat i evita conflictes d'adreces públiques.

Màscares de red y càlculo

Conceptes bàsics

La màscara define la parte de la IP que corresponde a la red. Ejemplos comunes:

  • /8 → 255.0.0.0 (Classe A, grans xarxes).
  • /16 → 255.255.0.0 (Classe B, xarxes mitjanes).
  • /24 → 255.255.255.0 (Classe C, xarxes petites).

Exemple de càlcul

IP 192.168.10.0/24 → red: 192.168.10.0, broadcast: 192.168.10.255, hosts: 192.168.10.1–192.168.10.254 (254 hosts).

Utilitza eines com ipcalc o calculadores en línia per simplificar el càlcul de subxarxes. Això és especialment útil en xarxes grans amb múltiples subxarxes.

VLAN — Què és i per què serveix

VLAN = Virtual LAN. Segmenta una mateixa xarxa física en múltiples segments lògics per separar trànsit, millorar seguretat i gestió.

  • Separar veu (VLAN 10) i dades (VLAN 20).
  • Separar departaments: administració, R+D, convidats.
  • Millorar el rendiment reduint el trànsit de difusió.

Les VLANs es configuren en switches gestionables i requereixen etiquetatge (802.1Q) per a trànsit entre switches. Això permet una millor organització i escalabilitat.

Xarxa en Màquines Virtuals

Modes d'adaptador

  • NAT: La VM comparteix la IP de l'amfitrió; accés sortint a Internet.
  • Bridged: La VM apareix a la xarxa física amb la seva pròpia IP.
  • Host-only: Xarxa privada entre host i VMs (sense Internet per defecte).
  • Internal: Xarxa només entre VMs.

Configura el mode segons les necessitats de la teva xarxa i seguretat. Per exemple, utilitza "Host-only" per a entorns de prova sense risc d'exposició externa.

Configuració de xarxa en Windows

GUI (Windows 10/11)

  1. Obrir Configuració → Xarxa i Internet → Ethernet/Wi‑Fi.
  2. Canviar les propietats de l'adaptador → Assignar IP manualment.

PowerShell (assignar IP estàtica)

# Trobar el nom de l'adaptador
Get-NetAdapter

# Assignar IP
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.1.50 -PrefixLength 24 -DefaultGateway 192.168.1.1

# Assignar DNS
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses ("8.8.8.8","8.8.4.4")

Utilitza PowerShell per automatitzar configuracions en múltiples dispositius, especialment en entorns empresarials.

Configuració de xarxa en Linux (Debian i Ubuntu 24.04)

Debian (ifupdown — /etc/network/interfaces)

# Exemple /etc/network/interfaces
auto eth0
iface eth0 inet static
    address 192.168.10.20
    netmask 255.255.255.0
    gateway 192.168.10.1
    dns-nameservers 8.8.8.8 8.8.4.4

Ubuntu 24.04 (netplan)

# /etc/netplan/01-network.yaml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [192.168.10.30/24]
      gateway4: 192.168.10.1
      nameservers:
        addresses: [8.8.8.8,8.8.4.4]

# Aplicar
sudo netplan apply

Utilitza comandes com ip a i ping per verificar la configuració. Això ajuda a identificar problemes de connectivitat.
