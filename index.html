Clasificación de IPs

Clases y rangos importantes

  • Clase A: 1.0.0.0 – 126.255.255.255 (grandes redes, hasta 16 millones de hosts).
  • Clase B: 128.0.0.0 – 191.255.255.255 (redes medianas, hasta 65,000 hosts).
  • Clase C: 192.0.0.0 – 223.255.255.255 (redes pequeñas, hasta 254 hosts).

Rangos privados (uso habitual)

  • 10.0.0.0/8: Utilizado en grandes empresas para redes internas.
  • 172.16.0.0/12: Ideal para redes medianas o departamentales.
  • 192.168.0.0/16: Común en redes domésticas y pequeñas oficinas.

Las direcciones privadas no son accesibles desde Internet y requieren NAT para conexiones externas.

Máscaras de red y cálculo

Conceptos básicos

La máscara define la parte de la IP que corresponde a la red. Ejemplos comunes:

  • /8 → 255.0.0.0 (Clase A, grandes redes).
  • /16 → 255.255.0.0 (Clase B, redes medianas).
  • /24 → 255.255.255.0 (Clase C, redes pequeñas).

Ejemplo de cálculo

IP 192.168.10.0/24 → red: 192.168.10.0, broadcast: 192.168.10.255, hosts: 192.168.10.1–192.168.10.254 (254 hosts).

Utiliza herramientas como ipcalc o calculadoras en línea para simplificar el cálculo de subredes.

VLAN — Qué es y para qué sirve

VLAN = Virtual LAN. Segmenta una misma red física en múltiples segmentos lógicos para separar tráfico, mejorar seguridad y gestión.

  • Separar voz (VLAN 10) y datos (VLAN 20).
  • Separar departamentos: administración, I+D, invitados.
  • Mejorar el rendimiento reduciendo el tráfico de difusión.

Las VLANs se configuran en switches gestionables y requieren etiquetado (802.1Q) para tráfico entre switches.

Red en Máquinas Virtuales

Modos de adaptador

  • NAT: La VM comparte la IP del anfitrión; acceso saliente a Internet.
  • Bridged: La VM aparece en la red física con su propia IP.
  • Host-only: Red privada entre host y VMs (sin Internet por defecto).
  • Internal: Red solo entre VMs.

Configura el modo según las necesidades de tu red y seguridad.

Configuración de red en Windows

GUI (Windows 10/11)

  1. Abrir Configuración → Red e Internet → Ethernet/Wi‑Fi.
  2. Cambiar las propiedades del adaptador → Asignar IP manualmente.

PowerShell (asignar IP estática)

# Encontrar el nombre del adaptador
Get-NetAdapter

# Asignar IP
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.1.50 -PrefixLength 24 -DefaultGateway 192.168.1.1

# Asignar DNS
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses ("8.8.8.8","8.8.4.4")

Utiliza PowerShell para automatizar configuraciones en múltiples dispositivos.

Configuración de red en Linux (Debian y Ubuntu 24.04)

Debian (ifupdown — /etc/network/interfaces)

# Ejemplo /etc/network/interfaces
auto eth0
iface eth0 inet static
    address 192.168.10.20
    netmask 255.255.255.0
    gateway 192.168.10.1
    dns-nameservers 8.8.8.8 8.8.4.4

Ubuntu 24.04 (netplan)

# /etc/netplan/01-network.yaml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [192.168.10.30/24]
      gateway4: 192.168.10.1
      nameservers:
        addresses: [8.8.8.8,8.8.4.4]

# Aplicar
sudo netplan apply

Utiliza comandos como ip a y ping para verificar la configuración.
