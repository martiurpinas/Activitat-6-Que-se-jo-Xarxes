Clasificación de IPs

↑ Inicio

Las direcciones IP se clasifican en diferentes clases según su rango. A continuación encontrarás una explicación detallada con ejemplos de direcciones IP.

Clases y rangos importantes

  • Classe A: 0.0.0.0 a 127.0.0.0 – Mascara: 255.0.0.0
  • Classe B: 128.0.0.0 a 191.0.0.0 – Mascara: 255.255.0.0
  • Classe C: 192.0.0.0 a 223.0.0.0 – Mascara: 255.255.255.0
  • Classe D: 224.0.0.0 a 239.0.0.0 - Mascara: No tiene una mascara de subred predeterminda.
  • Classe E: 240.0.0.0 a 255.0.0.0 - Mascara: Esta clase de IPs esta reservado para el uso experimental y usos de investigación.

Rangos privados (uso habitual):

  • Clase A: 10.0.0.0 - Mascara: /8
  • Clase B: 172.16.0.0 hasta 172.31.0.0 - Mascara: /16
  • Clase C: 192.168.0.0 - Mascara: /24

Máscaras de red

← IPs → VLAN

Las máscaras de red son fundamentales para entender cómo se divide una dirección IP entre la parte de red y la parte de host. A continuación encontrarás una explicación detallada de las máscaras de red y cómo calcularlas.

Conceptos básicos:

La máscara define la parte de la IP que corresponde a la red. Ejemplos comunes:

  • /8 → 255.0.0.0
  • /16 → 255.255.0.0
  • /24 → 255.255.255.0

Ejemplo de cálculo:

IP 192.168.10.0/24 → xarxa: 192.168.10.0, broadcast: 192.168.10.255, hosts: 192.168.10.1–192.168.10.254 (254 hosts).

Subnetting (exemple /26)

/26 → 255.255.255.192 → blocs de 64 adreces (62 hosts útils). De 192.168.10.0/24 en /26 obtenim 4 subxarxes: 192.168.10.0, .64, .128, .192.

Ej: IP: 192.168.10.70/26
Máscara: 255.255.255.192
Red: 192.168.10.64
Primer host: 192.168.10.65
Último host: 192.168.10.126
Broadcast: 192.168.10.127

VLAN

← Màscaras → VMs

Las VLANs (Virtual Local Area Network) permiten segmentar una misma red física en múltiples segmentos lógicos. A continuación encontrarás una explicación de las VLANs y su configuración básica.

¿Qué son y para qué sirven?

VLAN = Virtual Local Area Network. Segmenta una misma red física en múltiples segmentos lógicos para separar tráfico, mejorar seguridad y gestión. Esto permite crear redes virtuales independientes dentro de una misma infraestructura física.

Configuración básica en Cisco (Packet Tracer)

Switch> enable
Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name VOICE
Switch(config)# interface FastEthernet0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

Configuración de trunks (enlaces entre switches)

Para conectar switches y permitir que múltiples VLANs pasen por el mismo enlace:

Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20

Red en Máquinas Virtuales

← VLAN → Windows

Las máquinas virtuales ofrecen diferentes modos de adaptador de red, cada uno con sus propias características y casos de uso. A continuación encontrarás una explicación de los diferentes modos de adaptador en las máquinas virtuales.

Modos de adaptador más utilizados:

  • Modo NAT: La VM comparte la IP del anfitrión; tiene acceso saliente a Internet. Es el modo más común para uso general.
  • Modo Bridged (puente): La VM aparece en la red física con su propia IP, como si fuera un dispositivo físico más en la red.
  • Modo Host-only: Red privada entre el host y las VMs (sin internet por defecto). Útil para pruebas aisladas.
  • Modo Internal: Red solo entre VMs, sin acceso al host ni a Internet. Ideal para crear redes de prueba completamente aisladas.

Ejemplo práctico: Configuración en VirtualBox

Para configurar el modo de red en VirtualBox:

  1. Selecciona la VM → Configuración → Red
  2. Elige el adaptador y selecciona el modo deseado (NAT, Bridged, Host-only, Internal)
  3. Para Host-only, puedes configurar una red privada en Archivo → Preferencias → Red

Configuración de red en Windows

← VMs → Linux

Esta guía te ayudará a asignar IPs estáticas en Windows, tanto mediante la interfaz gráfica como mediante PowerShell.

Método 1: Interfaz gráfica (Windows 10/11)

  1. Abrir Configuración → Red e Internet → Ethernet o Wi‑Fi.
  2. Hacer clic en el adaptador de red activo.
  3. Desplazarse hasta Configuración de IP y hacer clic en Editar.
  4. Seleccionar Manual y activar IPv4.
  5. Introducir la IP estática, máscara de subred, puerta de enlace y DNS.
  6. Guardar los cambios.

Método 2: PowerShell (línea de comandos)

Para asignar una IP estática mediante PowerShell:

# Encontrar el nombre del adaptador
Get-NetAdapter

# Asignar IP estática
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.1.50 -PrefixLength 24 -DefaultGateway 192.168.1.1

# Asignar servidores DNS
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses ("8.8.8.8","8.8.4.4")

Nota: Ejecuta PowerShell como administrador para realizar estos cambios.

Configuración de red en Linux

← Windows ↑ Inici

Esta guía te ayudará a asignar IPs estáticas en Linux, cubriendo tanto Debian (con ifupdown) como Ubuntu 24.04 (con netplan).

Método 1: Debian (ifupdown — /etc/network/interfaces)

Para sistemas basados en Debian que usan ifupdown:

# Editar el archivo de configuración
sudo nano /etc/network/interfaces

# Ejemplo de configuración:
auto eth0
iface eth0 inet static
    address 192.168.10.20
    netmask 255.255.255.0
    gateway 192.168.10.1
    dns-nameservers 8.8.8.8 8.8.4.4

# Reiniciar el servicio de red
sudo systemctl restart networking

Método 2: Ubuntu 24.04 (netplan)

Para sistemas que usan netplan (Ubuntu 18.04+):

# Editar el archivo de configuración
sudo nano /etc/netplan/01-network.yaml

# Ejemplo de configuración:
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [192.168.10.30/24]
      gateway4: 192.168.10.1
      nameservers:
        addresses: [8.8.8.8,8.8.4.4]

# Aplicar la configuración
sudo netplan apply

Verificar la configuración

Después de configurar, verifica que todo funcione correctamente:

  • ip a - Muestra las direcciones IP configuradas
  • ip route - Muestra la tabla de enrutamiento
  • ping 8.8.8.8 - Prueba la conectividad a Internet
